# Agent OS

This folder contains the persistent context your AI coding agents use to
understand **how we build things here**.

- `product/` — what we're building and why (mission, roadmap, stack, decisions)
- `standards/` — code style and best practices the agent must follow
- `specs/` — per-feature specs (generated by SpecKit's `/speckit.specify`)

When you start a new chat with Claude Code, this folder is what gives the
agent its memory across sessions. Keep it accurate and pruned.

## Install

```bash
# Follow the official Agent OS install instructions
# https://github.com/buildermethods/agent-os
```

## Usage rule of thumb

- Edit `product/` when goals or scope change.
- Edit `standards/` when team conventions evolve.
- Never hand-edit files inside `specs/<spec-folder>/` after the spec is locked
  — supersede with a new spec instead.
