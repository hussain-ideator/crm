# API contract

Base URL: `http://localhost:8000/api/`

## Authentication

JWT (Bearer). Obtain via `POST /api/auth/login/`. Refresh via
`POST /api/auth/refresh/`.

```http
Authorization: Bearer <access_token>
```

## Resource conventions

| Verb   | Path                            | Purpose                |
|--------|---------------------------------|------------------------|
| GET    | `/api/leads/`                   | List (paginated)       |
| POST   | `/api/leads/`                   | Create                 |
| GET    | `/api/leads/{id}/`              | Retrieve               |
| PUT    | `/api/leads/{id}/`              | Replace                |
| PATCH  | `/api/leads/{id}/`              | Partial update         |
| DELETE | `/api/leads/{id}/`              | Soft delete            |
| POST   | `/api/leads/{id}/convert/`      | Custom action          |

Replace `leads` with: `contacts`, `companies`, `deals`, `activities`,
`notes`, `attachments`, `users`, `roles`, `pipelines`, `stages`.

## List query params (all list endpoints)

| Param      | Example                       | Meaning                       |
|------------|-------------------------------|-------------------------------|
| `page`     | `?page=2`                     | Page number (1-based)         |
| `page_size`| `?page_size=50`               | Items per page (max 100)      |
| `ordering` | `?ordering=-created_at`       | Sort, `-` = desc              |
| `q`        | `?q=acme`                     | Full-text search across name + email + phone |
| `<field>`  | `?status=qualified`           | Filter by exact value         |
| `<field>__in` | `?status__in=new,contacted`| Filter by set                 |

## Response shapes

### List
```json
{
  "count": 142,
  "next": "http://.../api/leads/?page=3",
  "previous": "http://.../api/leads/?page=1",
  "results": [ { "...": "..." } ]
}
```

### Detail
```json
{ "id": 1, "first_name": "Asha", "...": "..." }
```

### Validation error (HTTP 400)
```json
{
  "first_name": ["This field is required."],
  "email": ["Enter a valid email address."]
}
```

### Top-level error (HTTP 401/403/404/500)
```json
{ "detail": "Authentication credentials were not provided." }
```

## Versioning

No URL versioning for v1. If breaking changes are needed later, introduce
`/api/v2/`. Until then: additive changes only on `/api/`.

## OpenAPI

`GET /api/schema/` — OpenAPI 3 YAML, generated by drf-spectacular.
`GET /api/docs/` — Swagger UI.
`GET /api/redoc/` — ReDoc UI.
